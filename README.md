# ANÁLISIS RECLAMACIONES PACIENTE POWER BI
# Descripción del Proyecto 
Este proyecto consiste en un análisis integral de la experiencia del usuario. El objetivo principal fue identificar patrones clave en las reclamaciones de los pacientes y priorizar áreas de mejora operativa mediante el uso de un dataset con información relevante de la experiencia del paciente. El proyecto demuestra una combinación de capacidad analítica, manejo avanzado de Power BI y un enfoque orientado a resultados de negocio.

# Tecnologías y Habilidades Power BI Desktop

◉ Power BI: Creación de un dashboard interactivo con un layout optimizado para UX (2 columnas, bordes de 8px y jerarquía visual clara).
◉ Power Query (ETL): Limpieza y transformación de datos, incluyendo la creación de campos derivados como el número de semana (Week Number) e integración de paleta corporativa.
◉ Análisis de Datos: Extracción de insights accionables a partir de datos crudos (262 reclamaciones totales).
Storytelling: Elaboración de un informe ejecutivo orientado a la toma de decisiones estratégicas.

# Hallazgos Clave (Insights)

◉ Basado en los datos del período de Enero a Marzo 2016:
Punto Crítico de Satisfacción: Solo el 40,0% de los pacientes considera que el tiempo de espera es aceptable, marcando este indicador como CRÍTICO para la gestión.
◉ Principales Drivers de Reclamación: El 47% de las reclamaciones se concentran en dos motivos: Tiempo de Espera (25,95%) y Trato Irrespetuoso (21,37%).
◉ Tendencia Temporal: Marzo registró el mayor volumen de quejas (139 reclamaciones), lo que representa un 35% más que en enero.
◉ Oportunidad de Mejora: El 74% de las reclamaciones se agrupan en solo 4 motivos, lo que permite realizar intervenciones de alto impacto en poco tiempo.

# Propuestas de Mejora Estratégica

◉ Digitalización de Procesos: Implementar aplicaciones de citas y auto check-in para disminuir la carga administrativa y reducir los tiempos de espera.
◉ Formación en Soft Skills: Realizar capacitaciones enfocadas en el trato amable y la gestión emocional para el personal que interactúa con el paciente.
◉ Planificación por Estacionalidad: Ajustar la capacidad del centro (personal y camas) basándose en el histórico de picos detectados en meses de alta demanda como marzo.
